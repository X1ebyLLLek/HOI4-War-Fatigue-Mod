# Строгие правила замены текста (Strict text replacement rules)
1. **Никаких автоматических (fuzzy) замен.** Всегда заменяйте код только после точного поиска с использованием точного посимвольного совпадения. 
2. Всегда просматривайте файл (view_file) перед изменением и копируйте целевые строки буквально. Учитывайте точные символы конца строки (\n).
3. Избегайте использования консольных команд вроде sed или скриптов для массовых автоматических замен, если есть риск зацепить соседний код. Работайте вручную, блок за блоком.
4. **ОБЯЗАТЕЛЬНОЕ ОБНОВЛЕНИЕ АРХИТЕКТУРЫ.** После выполнения любой задачи программирования или рефакторинга агент ОБЯЗАН:
   - Проверить актуальность файлов `.agent/workflows/mod_architecture.md` и `.agent/workflows/mod_logic_tree.txt`.
   - Если добавлены новые файлы, флаги, переменные, эвенты или изменены логические цепочки (Tree flow), агент обязан вручную обновить эти документы перед завершением разговора. Эти файлы — Ground Truth, они не имеют права устаревать.

Эти правила добавлены по требованию пользователя, чтобы избежать случайных поломок кода.
